# CS000 Day 1: Learning How to Learn & The Engineering Mindset

## Overview
The most important skill in self-study is knowing how to learn. Without it, you'll waste hundreds of hours.

## Why Most Self-Learners Fail
- They confuse passive consumption (watching, reading) with active skill-building.
- They don't embrace struggle as the signal of learning.
- They lack a system for spacing, recall, and practice.

## Active vs. Passive Learning
- **Passive**: reading, watching, listening. Feels productive. Low retention.
- **Active**: coding from scratch, explaining out loud, solving unseen problems. Feels hard. High retention.
- Aim for 80% active time.

## The Forgetting Curve & Spaced Repetition
- Memory decays exponentially if not reinforced.
- Review at intervals: 1 day, 3 days, 1 week, 1 month.
- Use Anki for flashcards (I'll provide cards for each topic).

## The Feynman Technique
- Explain a concept simply, as if to a child.
- If you can't, you don't truly understand it.
- Rubber-duck debugging is the programming version.

## Deliberate Practice
- Choose tasks just beyond your current ability.
- Get immediate feedback (compiler, tests, mentor).
- Intense focus, no distractions.
- Reflect after every session.

## The 30-Minute Struggle Rule
1. Work on a problem alone for 30 min.
2. Re-read error messages.
3. Explain code out loud.
4. Google exact error in quotes.
5. Ask with a minimal reproducible example.
6. Walk away 20 min.

## Setting Up Your Learning System
- **Pomodoro**: 25 min work / 5 min break. 4 cycles = long break.
- **Environment**: dedicated space, phone away, website blocker.
- **Notes**: one `.md` file per concept, linked, in your own words.
- **Daily log**: what you learned, struggled with, tomorrow's task.
- **Time tracking**: Toggl or spreadsheet.

## Key Mindset Shifts
- "I'm stuck" → "My brain is rewiring."
- "I finished a tutorial" → "Can I rebuild it from scratch?"
- "I don't get it" → "I need to find a simpler explanation."

## Exercises
1. Write your learning intentions in `day1-notes.md`.
2. Explain a known concept using Feynman technique.
3. Solve a new LeetCode Easy with the 30-min rule.
4. Set up your daily template.
5. Debug the binary search function via rubber-duck.
